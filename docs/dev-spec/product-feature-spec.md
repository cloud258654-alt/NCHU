## 1. 產品定位

本產品是一套給店家使用的智能商業與輿情分析服務。使用者不需要登入複雜後台，而是透過 Line Bot 提出需求，例如「幫我監測文章牛肉湯最近的網路評價」或「如果有負評請通知我」。系統會透過 n8n 自動化流程把請求轉成任務，後端建立爬蟲工作，收集網路貼文與留言，再透過機器學習與 AI 方法分析評論內容，最後產出警示通知、摘要報告與改善建議。

產品初期主打「輿情監測與負評警示」，後續擴充「線上預約管理」與「商業洞察分析」。

## 2. 目標使用者

| 使用者 | 情境 | 痛點 | 產品價值 |
|---|---|---|---|
| 餐飲店家老闆 | 想知道網路上有沒有人抱怨服務或餐點 | 沒時間每天看社群平台 | 自動收集評論並通知 |
| 店長或客服人員 | 需要即時知道負評並回應 | 負評發現太晚，容易擴散 | 即時警示與改善建議 |
| 小型品牌經營者 | 想了解顧客對產品與服務的看法 | 評論分散在多平台，難以統整 | 彙整正負評與議題 |
| 系統管理員 | 需要管理不同客戶的任務 | 任務與資料來源複雜 | 後台資料結構清楚可追蹤 |

## 3. 核心使用流程

使用者在 Line Bot 輸入服務請求，例如「請幫我監測文章牛肉湯最近的負評」。Line Bot 將使用者 Line ID、店家名稱、關鍵字與服務類型傳給 n8n。n8n 依照流程呼叫後端 API，建立 client、business、service_task 與 crawl_job。爬蟲模組依照平台與關鍵字收集資料，將貼文、留言與互動數據寫入資料庫。分析模組對內容進行情緒分類、關鍵議題整理與風險判斷。若出現高風險負評，系統會透過 Line Bot 發送警示。若是一般分析結果，系統會產生摘要報告與改善建議。

## 4. MVP 功能範圍

| 功能模組 | MVP 功能 | 不在 MVP 的功能 |
|---|---|---|
| Line Bot | 接收服務請求、回傳任務狀態、發送警示 | 複雜選單、完整會員中心 |
| n8n | 接收參數、呼叫後端 API、觸發爬蟲、發送通知 | 複雜多分支商業流程 |
| 後端 API | 建立客戶、店家、服務任務、爬蟲任務、查詢分析結果 | 完整 SaaS 後台 |
| 爬蟲 | 至少完成二至三個平台資料收集 | 所有平台完整支援 |
| 分析 | 正負面分類、摘要、警示判斷 | 高階預測模型、完整商業 BI |
| 報告 | 文字摘要、好負評統計、改善建議 | 完整 dashboard 與 PDF 報告 |
| 預約 | 暫不納入第一階段 | 第二階段開發 |

## 5. Line Bot 對話設計

| 使用者輸入 | 系統行為 | 回覆範例 |
|---|---|---|
| 我要監測店家評價 | 系統詢問店家名稱 | 請輸入要監測的店家名稱 |
| 文章牛肉湯 | 系統建立或查詢 business | 已收到店家名稱，請選擇要監測的平台 |
| Dcard / PTT / Google Maps | 系統建立 service_task | 已建立輿情監測任務 |
| 查詢報告 | 系統查詢最近分析結果 | 目前共收集 32 則評論，其中負評 6 則 |
| 停止監測 | 系統更新任務狀態 | 已停止此店家的輿情監測服務 |

## 6. 警示通知設計

當系統發現明顯負評、抱怨、服務問題、食品安全疑慮、價格爭議、排隊問題、態度問題或社群聲量異常上升時，應主動發送 Line 通知。

通知格式建議如下：

```text
輿情警示通知

店家：文章牛肉湯
平台：Dcard
風險類型：服務負評
情緒判斷：負面
內容摘要：有顧客反映尖峰時段等待時間過長，且現場動線混亂。
建議處理：建議店家檢查尖峰時段人力安排，並加強排隊與叫號流程。
原文連結：貼文 URL
