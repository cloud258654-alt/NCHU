# BI-RMP 智能商業與輿情分析服務

BI-RMP 以店家名稱作為服務核心，透過後端自動執行目前已支援的平台爬蟲，收集公開網路風評、貼文、評論與互動數據，再交由分析與警示流程處理。

目前 MVP 平台順序：

```text
ptt -> google_maps -> threads
```

`dcard` 先保留為下一優先級規劃，不再保留 active adapter 程式碼。`docs/design/Schema_overview.png` 與 `docs/design/Frontend.html` 只作為歷史設計參考，開發以 `docs/dev-spec/` 與 `database/schema.sql` 為準。

## Service Flow

```text
business_name + optional keyword
-> clients / business
-> service_tasks.config
-> crawl_jobs
-> crawl_posts (latest state + crawl_job_id) -> crawl_comments
-> post_metric_snapshots / comment_metric_snapshots
-> analysis_results
-> alerts
```

使用者或 n8n 不需要指定平台，也不需要提供 Google Maps URL。後端會先嘗試 URL discovery；若暫時找不到明確 Google Maps place URL，會改用店家名稱建立 Google Maps 搜尋 URL，再交給 Google Maps crawler 解析實際來源。

## Database

Active clean rebuild schema:

```text
database/schema.sql
```

新 schema 使用 `BIGINT GENERATED BY DEFAULT AS IDENTITY` 作為主鍵，不再使用 UUID 或 human-readable id 補丁。這份 SQL 會 drop runtime/legacy crawler tables，但不主動 drop `review%` tables 或 `master_reviews_enriched`。

核心表：

- `clients`
- `business`
- `service_tasks`
- `crawl_jobs`
- `crawl_posts`
- `crawl_comments`
- `post_metric_snapshots`
- `comment_metric_snapshots`
- `analysis_results`
- `alerts`

`crawl_posts` 是唯一保存 `crawl_job_id` 的 latest-state table。`crawl_comments` 只透過 `crawl_post_id` 連到貼文；metric snapshots 只保存 parent ID 與 `collected_at`。不建立 observation/event table 或 context view，店家資訊一律從 post 經 job、task 明確 JOIN 取得。

### Google Maps source discovery

Google Maps source discovery is URL-only and runs only when selected platforms include `google_maps`. Threads-only and PTT-only runs do not perform Google Maps URL discovery.

Discovery tries SearXNG, DuckDuckGo, and Bing according to configuration, scores candidate Google Maps URLs, and falls back to a generated Google Maps search URL when no reliable place URL is available.

Optional environment variables:

```env
GOOGLE_MAPS_DISCOVERY_ENABLED=true
GOOGLE_MAPS_DISCOVERY_ENGINE=auto
GOOGLE_MAPS_DISCOVERY_MAX_RESULTS=10
GOOGLE_MAPS_DISCOVERY_QUERY_VARIANT_LIMIT=4
SEARXNG_BASE_URL=http://localhost:8080
```

`SEARCH_ENGINE=auto` and `GOOGLE_MAPS_DISCOVERY_ENGINE=auto` try `searxng -> duckduckgo -> bing` when `SEARXNG_BASE_URL` is set, otherwise `duckduckgo -> bing`. Use `disabled` for Google Maps discovery to skip search engines and use the generated Google Maps search URL fallback.

Threads crawling can use a manually generated Playwright login session:

```env
THREADS_STORAGE_STATE_PATH=storage/threads_state.json
```

The storage-state file is local runtime state and must not be committed.

Accepted source URLs include Google Maps place URLs, `cid` URLs, `place_id` / `query_place_id` URLs, and Maps search URLs. Discovery diagnostics include candidate score, candidate type, rejection reason, selected source, and fallback status. Search snippets are not treated as Google Maps review content.

## Public Runner Options

| Option | 說明 |
| --- | --- |
| `--business-name` | 必填。目標店家名稱，也是風評搜尋基礎詞。 |
| `--keyword` / `--purpose` | 選填。額外監測意圖，例如服務態度、排隊、價格。 |
| `--client-name` / `--client-id` | 開發或 demo 用，正式 Line Bot 可由使用者資料帶入。 |
| `--business-id` | 已知既有店家 id 時可帶入。 |
| `--line-user-id` | Line Bot 來源可帶入。 |
| `--service-type` | `reputation_monitoring`、`reservation_management`、`business_insight`。 |
| `--schedule-type` | `once`、`hourly`、`daily`、`weekly`。 |
| `--max-scroll` / `--max-minutes` / `--max-results` | 開發測試限制。 |
| `--dry-run` | 不寫入爬蟲資料。 |

`--platform`、`--url`、`--headless`、`--date-range`、`--fetch-comments`、search engine 相關參數不再是 public runner 介面；需要時由各平台 adapter 或內部 service 自行管理。
